B2B SaaS Renewal Prediction Report

Objective: Build classification models to predict renewed_next_quarter and compare KNN with Decision Tree from both performance and business perspectives.

Step 1. Data Loading and Preprocessing

  • The dataset was loaded from a CSV file.
  • The categorical variables industry, region, support_level, and contract_type were encoded using LabelEncoder.
  • account_id and the target variable renewed_next_quarter were excluded from the feature set.
  • The remaining 20 variables were used as model inputs.
  • The data was split into 80% training and 20% testing sets using random_state = 42.
  • For KNN, a StandardScaler version was prepared because KNN is distance-based.
  • For Decision Tree, the original unscaled data was used.

Data Imbalance Check

The class distribution was close to balanced, so model performance is not heavily biased by class imbalance. This makes the comparison between models more meaningful.

Step 2. KNN Model Training and Evaluation

The KNN model was trained with KNeighborsClassifier(n_neighbors=5).

Metric KNN (k=5)
Accuracy 0.7085
Precision 0.7111
Recall 0.7085
F1-Score 0.7068

Class-level performance:

  • Class 0: Precision 0.73 / Recall 0.63 / F1 0.68
  • Class 1: Precision 0.69 / Recall 0.78 / F1 0.73

Interpretation

KNN is a distance-based, non-parametric method. It predicts customer renewal by comparing each customer to nearby observations in the feature space. Its strength is that it can capture local similarity, but its weakness is that performance may deteriorate in higher-dimensional settings because distances become less informative, which is known as the curse of dimensionality.

Step 3. Decision Tree Model Training and Evaluation

The Decision Tree model was trained with DecisionTreeClassifier(max_depth=5, random_state=42).

Metric Decision Tree (max_depth=5)
Accuracy 0.7195
Precision 0.7225
Recall 0.7195
F1-Score 0.7178

Class-level performance:

  • Class 0: Precision 0.75 / Recall 0.64 / F1 0.69
  • Class 1: Precision 0.70 / Recall 0.79 / F1 0.74

Interpretation

Decision Tree is a rule-based and interpretable model. It predicts renewal by splitting customers into groups according to explicit conditions. This is useful in a business setting because the model can show which rules separate likely renewals from likely churn. At the same time, trees can overfit if they become too deep. Setting max_depth = 5 helps control model complexity and improves generalization.

Step 4. Apple-to-Apple Comparison and Final Interpretation

Overall conclusion: The Decision Tree is superior across all four key evaluation metrics.

Final Reporting Points

  • Numerical comparison
    • Decision Tree outperformed KNN in Accuracy, Precision, Recall, and F1-Score.
    • The performance gap is not extremely large, but it is consistent across all evaluation metrics.
    • This consistency suggests that Decision Tree is more reliable overall for this dataset.
  • Business interpretation: Recall vs. F1-score
    • In a customer renewal prediction problem, Recall is especially important when the company wants to identify as many customers at risk of non-renewal as possible.
    • If the model misses a true non-renewal customer (False Negative), the company may fail to intervene in time, which can directly lead to lost revenue.
    • However, focusing only on Recall may increase unnecessary interventions on customers who would have renewed anyway (False Positive), which wastes customer success resources.
    • Therefore, F1-score is a more balanced metric for model selection because it captures the trade-off between Precision and Recall.
  • Model recommendation
    • From a performance perspective, the Decision Tree is the stronger model.
    • From an interpretability perspective, the Decision Tree is also preferable because its rule-based structure is easier to explain to management and operational teams.
    • Therefore, the final recommendation is to adopt the Decision Tree as the primary model for customer renewal prediction in this B2B SaaS context.
    • KNN can still be used as a benchmark model, but it is less suitable as the main business decision-support tool.

Executive Summary

  • The dataset was preprocessed appropriately, with categorical variables encoded and separate scaled and unscaled versions prepared for KNN and Decision Tree.
  • KNN showed reasonable predictive ability, but it remained weaker than the Decision Tree on all main metrics.
  • Decision Tree achieved the best overall balance of predictive performance and explainability.
  • For this business problem, F1-score is the most practical summary metric for final model comparison, while Recall remains strategically important because missing at-risk customers is costly.
  • The recommended final model is Decision Tree.
