Model Comparison Report

Topic: Predicting Customer Renewal in B2B SaaS

1. Dataset Overview

The dataset contains 10,000 customer accounts and 22 columns. The target variable is renewed_next_quarter, which indicates whether a customer renewed in the next quarter.

Target Class Count Ratio
1 (Renewed) 5,138 51.38%
0 (Churned) 4,862 48.62%

This means the target classes are relatively balanced. Therefore, model performance must come from real pattern learning, not class bias.

2. Models Compared

  • Classification Tree
  • K-Nearest Neighbors (KNN)

The best K value for KNN was 9.

3. Main Result

Final Best Model: Classification Tree

Even though KNN improved with tuning (K=9), the tree model still matched the actual outcomes better.

4. Why Classification Tree Performed Better

Nonlinear decision structure

The dataset likely contains nonlinear patterns (e.g., threshold-based behavior), which trees capture well.

Threshold-based business logic

Customer renewal often depends on clear cutoffs (e.g., usage drop, engagement level), not just similarity.

Interpretability

The tree provides explicit decision rules, which are directly usable in business decision-making.

5. Why K = 9 Worked Best for KNN

  • Small K → too sensitive to noise
  • Large K → too averaged (loses meaning)
  • K = 9 → balanced noise reduction + local pattern 유지

However, KNN still underperformed because similarity alone was not enough to explain the data.

6. Interpretation

KNN answers: "Who is similar?"

Tree answers: "What rule separates outcomes?"

This dataset favors rule-based segmentation rather than similarity-based prediction.

7. Business Insight

  • Tree → better for explaining churn drivers
  • Better for actionable strategy (CS team 활용 가능)
  • KNN → 참고 모델 수준

8. Final Conclusion

The Classification Tree is the most appropriate model for this problem because it captures decision rules and aligns better with actual outcomes.
